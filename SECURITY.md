# Security Policy

## Supported Versions

Security fixes are applied to the current maintained release line.

| Version | Supported |
|---------|-----------|
| `2026.718.1` | Yes |
| Earlier releases | No |

Update with `openacp update` before reporting an issue that may already be
fixed in the latest release.

## Reporting a Vulnerability

Report vulnerabilities privately through [GitHub Private Vulnerability
Reporting](https://github.com/an1creator/OpenACP/security/advisories/new).

Do not open a public issue, discussion, or pull request for an unpatched
vulnerability. Never publish bot tokens, API keys, proxy credentials, session
data, private source code, or unredacted logs.

Include the affected OpenACP version, deployment environment, impact, minimal
reproduction steps, and any safe remediation ideas. Use placeholders for every
secret.

If the private report form is temporarily unavailable, contact the repository
owner through the current method on the [`an1creator` GitHub
profile](https://github.com/an1creator) and share only a high-level summary
until a private channel is established.
